# Stack Dependencies - Mixed Stack + Unit Example

Based on [RFC #5663](https://github.com/gruntwork-io/terragrunt/issues/5663) - Stack Dependencies via Autoinclude.

## What this demonstrates

A single `autoinclude` block declaring two dependencies in parallel:

- One dependency on a unit inside a nested `stack` (`stack.foo`)
- One dependency on a sibling `unit` (`unit.baz`)

Demonstrates that `stack.<stack_name>.<unit_name>.path` and `unit.<name>.path` references can coexist in the same autoinclude block, and that the DAG correctly orders all three (provider, baz) before the consumer (bar).

## Directory structure

```
mixed-stack-unit/
  catalog/
    stacks/
      foo/
        terragrunt.stack.hcl   # nested stack: defines unit "provider"
    units/
      provider/                # produces output "val"
      baz/                     # produces output "val"
      bar/                     # consumes both
  live/
    terragrunt.stack.hcl       # stack "foo" + unit "baz" + unit "bar"
```

## Stack file

`live/terragrunt.stack.hcl`:

```hcl
stack "foo" {
  source = "../catalog/stacks/foo"
  path   = "foo"
}

unit "baz" {
  source = "../catalog/units/baz"
  path   = "baz"
}

unit "bar" {
  source = "../catalog/units/bar"
  path   = "bar"

  autoinclude {
    dependency "provider" {
      config_path = stack.foo.provider.path
      mock_outputs_allowed_terraform_commands = ["validate", "plan"]
      mock_outputs = { val = "fake-from-stack" }
    }

    dependency "baz" {
      config_path = unit.baz.path
      mock_outputs_allowed_terraform_commands = ["validate", "plan"]
      mock_outputs = { val = "fake-from-unit" }
    }

    inputs = {
      val_from_stack = dependency.provider.outputs.val
      val_from_unit  = dependency.baz.outputs.val
    }
  }
}
```

## How to run

```bash
cd live
terragrunt stack generate --experiment stack-dependencies
terragrunt run --all --experiment stack-dependencies -- apply -auto-approve
```

Expected generated layout:

```
live/.terragrunt-stack/
  foo/
    provider/
  baz/
  bar/
    terragrunt.autoinclude.hcl
```

Expected `terragrunt.autoinclude.hcl` for `bar`:

```hcl
# Generated by Terragrunt. Do not edit manually.
dependency "provider" {
  config_path                             = "../foo/provider"
  mock_outputs_allowed_terraform_commands = ["validate", "plan"]
  mock_outputs = {
    val = "fake-from-stack"
  }
}

dependency "baz" {
  config_path                             = "../baz"
  mock_outputs_allowed_terraform_commands = ["validate", "plan"]
  mock_outputs = {
    val = "fake-from-unit"
  }
}

inputs = {
  val_from_stack = dependency.provider.outputs.val
  val_from_unit  = dependency.baz.outputs.val
}
```

Expected apply order:
1. `foo/provider` and `baz` apply in parallel (no inter-dependency)
2. `bar` applies last, receiving `from-stack-foo` and `from-sibling-baz`

Validate:

```bash
cat .terragrunt-stack/bar/.terragrunt-cache/*/input.txt
# Expected: stack=from-stack-foo unit=from-sibling-baz
```

## Clean up

```bash
terragrunt stack clean
```
