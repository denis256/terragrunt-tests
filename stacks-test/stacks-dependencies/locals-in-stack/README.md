# Stack Dependencies - Locals in Stack Example

Based on [RFC #5663](https://github.com/gruntwork-io/terragrunt/issues/5663) - Stack Dependencies via Autoinclude.

## What this demonstrates

`locals` defined at the top of a `terragrunt.stack.hcl` file are usable everywhere in that file, including:

- `source` and `path` on `stack` and `unit` blocks
- `dependency.config_path` inside `autoinclude`
- `mock_outputs` inside `autoinclude`
- `inputs` inside `autoinclude`

In the prototype, locals are evaluated and merged into the eval context **before** the autoinclude bodies are resolved, so `local.*` references inside autoinclude are resolved at stack generation time and baked into the generated `terragrunt.autoinclude.hcl` as literals. The unit never has to re-evaluate the surrounding stack's locals.

(The RFC sketch initially proposed banning `local.*` inside autoinclude. That proposal did not land - locals are permitted throughout. See `internal/hclparse/parse.go:78-95`.)

## Directory structure

```
locals-in-stack/
  catalog/
    stacks/
      foo/
        terragrunt.stack.hcl   # nested stack: defines unit "provider"
    units/
      provider/                # produces output "val"
      consumer/                # consumes val and env
  live/
    terragrunt.stack.hcl       # uses locals everywhere
```

## Stack file

`live/terragrunt.stack.hcl`:

```hcl
locals {
  env          = "test"
  stack_source = "../catalog/stacks/foo"
  consumer_src = "../catalog/units/consumer"
  stack_path   = "foo"
  mock_val     = "mock-${local.env}"
}

stack "foo" {
  source = local.stack_source
  path   = local.stack_path
}

unit "consumer" {
  source = local.consumer_src
  path   = "consumer"

  autoinclude {
    dependency "provider" {
      config_path = stack.foo.provider.path

      mock_outputs_allowed_terraform_commands = ["validate", "plan"]
      mock_outputs = {
        val = local.mock_val          # local.* resolved at generation time
      }
    }

    inputs = {
      env = local.env                  # local.* resolved at generation time
      val = dependency.provider.outputs.val   # dependency.* preserved for runtime
    }
  }
}
```

## How to run

```bash
cd live
terragrunt stack generate --experiment stack-dependencies
terragrunt run --all --experiment stack-dependencies -- plan
terragrunt run --all --experiment stack-dependencies -- apply -auto-approve
```

Expected `terragrunt.autoinclude.hcl` for `consumer` (locals baked in as literals):

```hcl
# Generated by Terragrunt. Do not edit manually.
dependency "provider" {
  config_path                             = "../foo/provider"
  mock_outputs_allowed_terraform_commands = ["validate", "plan"]
  mock_outputs = {
    val = "mock-test"
  }
}

inputs = {
  env = "test"
  val = dependency.provider.outputs.val
}
```

Validate:

```bash
cat .terragrunt-stack/consumer/.terragrunt-cache/*/input.txt
# Expected: env=test val=Hello-from-provider
```

## Resolution timing

| Reference                              | When resolved          | How it appears in generated file |
|----------------------------------------|------------------------|----------------------------------|
| `local.env`, `local.mock_val`          | stack generation       | literal value                    |
| `stack.foo.provider.path`              | stack generation       | relative path string             |
| `dependency.provider.outputs.val`      | unit apply (runtime)   | preserved verbatim               |
